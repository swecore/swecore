# Demo Workflow

The SWECore public demo should use sample data only. It is meant to show a
guided review loop, not to expose private repositories, production systems, or
sensitive implementation details.

No hosted control plane is required for the early-access demo. The demo should
avoid any claim that the full SWECore runtime is publicly shipped.

## Workflow steps

1. Select repo.
2. Scan workspace.
3. Review findings.
4. Generate change plan.
5. Decision panel.
6. Receipt.
7. Handoff.

## Demo boundary

The public demo is illustrative. It does not prove the full SWECore runtime is
publicly shipped.
