# Data Boundary

## Early-access data boundary

Early-access intake should stay low-sensitivity. Visitors should not submit
private code, credentials, customer records, or production diagnostics.

## What we may ask

The waitlist may ask for:

- name and email;
- role and company/team-size bucket;
- current AI coding tools;
- preferred OS/development environment;
- optional update consent, if shown.

## What we do not ask

The waitlist does not ask for:

- private repository URLs;
- source code;
- API keys, secrets, or access tokens;
- customer data;
- production logs;
- GitHub, IDE, or model-provider account access.

## No raw PII in repo

Raw waitlist rows, contact exports, names, email addresses, and other personally
identifying data should not be committed to this repository.

## No third-party marketing sharing

SWECore public materials should not promise or implement third-party marketing
sharing of waitlist data.

## Service providers may operate the website/waitlist flow

Service providers may operate the website or waitlist flow. That does not change
the rule that users should not submit source code, secrets, customer data,
production logs, or private repository URLs through the waitlist.
