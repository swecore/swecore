# What Is SWECore?

## What SWECore is

SWECore is a local-first governance and evidence layer for AI-assisted software
delivery.

It helps make AI-assisted repository work easier to review before it is trusted.
The focus is scope, evidence, receipts, known limits, and reviewer handoff.

## What SWECore is not

SWECore is not a coding agent, IDE, model wrapper, prompt marketplace, or hosted
dashboard today. It does not replace a developer, reviewer, test suite, issue
tracker, or security program.

It is also not production-ready or compliance-ready today.

## Why this matters now

AI-generated code changes can happen quickly. Review, scope control, evidence,
and claim boundaries often fall behind.

That creates a gap: a change may look complete in chat, but the reviewer still
needs to know what changed, what was tested, what was not proven, and what
should happen next.

SWECore is designed around that gap.

## Where SWECore fits around existing tools

Teams can keep using their existing coding agents, editors, model providers, and
review workflows. SWECore sits around that work as a governance layer.

It helps answer practical review questions:

- What is the intended scope?
- What evidence exists?
- What claim is not supported yet?
- What should the reviewer inspect?
- What is the next safe action?

## Current early-access boundary

Early access is focused on product evaluation, public documentation, demo
materials, trust boundaries, and low-sensitivity waitlist intake.

This repository is not the full commercial runtime and should not be treated as
an installable product release unless public runtime artifacts are separately
shipped and proven.
